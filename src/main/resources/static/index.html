How Callisto works

Three deterministic stages. No hallucination. Every value is traceable to a pixel on the page.

1

PDF preprocessing

PDFBox splits multi-page documents. Each page is passed to Azure Document Intelligence using the prebuilt-document model. Pages are processed in parallel via Java 21 virtual threads — a 6-page 1040 finishes in the time it takes to process one page.

2

KV normalisation

Azure DI returns verbose key descriptions ("Wages, salaries, tips, etc. Attach Form(s) W-2"). A three-stage resolver — whitespace normalisation → punctuation stripping → snake_case fallback — maps every raw key to a canonical field name deterministically. No LLM involved.

3

Reconciliation & severity scoring

Extracted values are compared field-by-field against reference values you supply (from your own system of record). Delta magnitude drives severity: HIGH (>5%), MEDIUM (1–5%), LOW (<1%). Cross-document discrepancies are flagged at the file level.
